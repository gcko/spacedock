A few common workflows should exist as examples, so we don't reinvent them when commissioning:

- development
- experiment
- daily routine
- working with CRM

The commission skill (`skills/commission/SKILL.md`) currently walks the captain through a from-scratch design conversation. When a captain wants something common, they re-derive the same structure each time. Surfacing a handful of well-shaped examples (or templates) at the right moment in the commission flow would shortcut that.

Ideation should brainstorm with the captain on:
- which workflow shapes are common enough to warrant first-class examples,
- how they should be surfaced in commission (suggested at intake? listed in a menu? loaded as a starting template?),
- what the example looks like for each shape (entity-label, stages, gates, mods, etc.),
- whether examples ship in-tree (under `examples/`?) or live as references in the commission skill,
- and how to keep examples from rotting as the workflow conventions evolve.
