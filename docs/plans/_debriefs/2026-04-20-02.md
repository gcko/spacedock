## Issues — Workflow

- **`pr:` field not auto-stamped when a PR exists before the pr-merge hook fires.** Caught retroactively on #210 (PR #143 had merged externally but `pr:` was empty, so the pr-merge startup scan missed it; FO had to manually stamp `pr: #143` mid-session) and on #211 (PR #142 was open before this session booted, same cause). Pattern: the `claude-team build` dispatch path doesn't stamp `pr:` at PR-create time; the pr-merge mod hook only stamps at merge boundary. - **The full pr-merge loop works end-to-end.** Three PR-based ships (#211, #212, #187-external) all flowed through mod-block → push approval → PR creation → CI approval → merge → done → archive → cleanup. The mechanism-level guards (`status --set` refusing terminal transitions without mod-block cleared) fired correctly. The `--force` bypass is captain-authorized and well-scoped.

The output was high-quality (caught the regex group-10 ambiguity, questioned the protocol framing). The contract gap is whether this pattern is sanctioned or emergency-mode.
- **#187 cycle-1 REJECT was the right call.** Catching circumstantial-vs-direct evidence at the gate saved us from merging on a flake-assumption that could have been a stable failure. The cost was one extra cycle (~7 min impl); the value was confidence.
- **Codex CI needs a "local-reproducible" story.** Validator on #187 spot-checked via prompt-replay rather than live codex run because macOS TCC sandboxing blocked local codex. Not a session-blocker, but an ergonomics debt.
- **CI cost-per-run has settled at ~$1-3.** All three #211 iterations plus #212 fit within this envelope; no $2-per-file budget caps hit this session.
